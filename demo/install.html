统一安装入口

Elens 现在按 Web、Extension、Desktop 三类宿主公开官方入口。对外是清晰的安装方式，对内仍然复用同一份 inspector core，并把 viewport、clipboard、capture 等宿主能力收口到 adapter/runtime 层。

Web

站点 / Demo / SDK 挂载

直接通过 `elens/web` 获取 `mountElementInspector`，适合普通站点、嵌入式调试页和 demo 环境。

import { mountElementInspector } from 'elens/web'

mountElementInspector({ enabled: true })
  • 默认入口最轻量
  • 可按需接 viewport controller
  • 当前 demo 已实际走这条路径
Extension

Chrome 扩展接入

通过 `elens/extension` 暴露 bridge controller 和 content script 薄入口，让窗口控制与截图能力留在宿主适配层。

import {
  createChromeExtensionViewportController,
  mountChromeExtensionInspector,
} from 'elens/extension'
  • 构建链继续复用现有 extension build
  • content script 只负责消息装配
  • 更适合后续统一安装页直接指向扩展版本
Desktop

Electron / Tauri 骨架

通过 `elens/desktop` 暴露 desktop controller stub，先把真实桌面宿主的接线方式固定下来，再逐步补完整桥接。

import {
  createDesktopViewportController,
  mountElementInspector,
} from 'elens/desktop'
  • 这轮是可接入骨架，不是完整桌面分发
  • 已补最小 Electron 示例文件
  • 后续重点是 window bounds / clipboard / capture

接下来怎么继续

  1. 统一安装入口页继续保留三种官方路径，不再把 Chrome extension 当唯一产品面。
  2. Web 入口保持最简挂载体验，作为默认 SDK 使用方式。
  3. Extension 入口继续承接浏览器能力桥接，并维持当前构建链。
  4. Desktop 入口下一阶段补 Electron/Tauri 真桥接，而不是继续散落在 `mount.ts` 里写条件分支。
